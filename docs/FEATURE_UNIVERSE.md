# FEATURE_UNIVERSE — World-class commerce capabilities

> A cited map of every meaningful capability a modern e-commerce platform can have, organised by domain. Each entry tags a **tier** (table-stakes / competitive / visionary), names **exemplars**, notes **Malaysia/SEA relevance**, and gives a **build note** for *this* stack (Laravel 13 · Livewire 4 · SQLite/MySQL · Scout · brick/money). Researched against platform leaders (Shopify, Amazon, Stripe, BigCommerce; composable: Medusa, Saleor, Vendure, commercetools), SEA/Malaysia (Shopee, Lazada, TikTok Shop; FPX/DuitNow/e-wallets/BNPL; SST + LHDN MyInvois; EasyParcel/couriers), and AI-native commerce shipping in 2025–26.
>
> Compiled by a 14-agent research+audit workflow. Sources are inline. Cross-referenced against the current build in `ECOMMERCE_AUDIT.md`; gaps tiered in `GAP_ANALYSIS.md`.

## 1. Catalog & product data

- **Variant/option matrix** *(table-stakes)* — products expand into per-SKU variants with price/weight/stock. Shopify, Vendure, Saleor, commercetools. *Already mandated here (Hard Rule 6).* — vendor docs: `docs.vendure.io/guides/core-concepts/products/`
- **Metafields / metaobjects (flexible custom data)** *(competitive)* — arbitrary typed fields without migrations (halal-cert no., SIRIM code, ingredients, expiry). Shopify metafields/metaobjects, Saleor/Medusa metadata, commercetools custom types. *Build:* polymorphic `metafields` table or spatie schemaless-attributes JSON; index searchable keys in Scout. — `help.shopify.com/.../category-metafields`
- **Standard taxonomy + category attributes** *(competitive)* — canonical category tree, each category unlocking typed facets. Shopify Standard Product Taxonomy, Amazon browse nodes. *Build:* seed tree + per-category attribute defs; dynamic Livewire form inputs.
- **Product bundles & kits** *(competitive)* — sellable composite priced as a unit, decrementing each component's stock. Shopify Bundles, BigCommerce, commercetools. *Build:* `bundle_components` table; expand to components inside the atomic checkout lock.
- **Enhanced / A+ rich content** *(competitive)* — brand-controlled media blocks beyond plain description. Amazon A+, Shopify sections. *Build:* block-based JSON schema rendered via whitelisted Blade components; gate behind verified/Mall tier. — `bebolddigital.com/blog/amazon-seller-central`
- **Ratings & reviews with photo/video + verified-purchase + seller reply** *(table-stakes)* — Amazon, Shopee (coins-for-review), Yotpo. *Build:* review tied to `order_item` (verified), spatie/medialibrary media, aggregate cached + indexed in Scout. — `miracuves.com/blog/what-is-shopee-and-how-it-work/`

## 2. Search & discovery

- **Faceted search + typo tolerance + synonyms + instant autocomplete** *(table-stakes)* — Algolia, Meilisearch, Saleor. *MY:* mixed BM/English/Manglish queries. *Build:* Scout→Meilisearch typo/synonyms/filterable attributes; Livewire instant-search. — `meilisearch.com/blog/meilisearch-vs-algolia`
- **Search merchandising & ranking rules** *(competitive)* — pin/promote/bury, boost by margin/recency, query redirects. Algolia Merchandising Studio, commercetools. *Build:* query→rule mappings applied as Scout ranking overlays; admin CRUD. — `americaneagle.com/.../algolia-best-practices`
- **Semantic / vector (hybrid) search** *(competitive)* — embedding search understanding intent + keyword hybrid; +8–20% conversion. Meilisearch vector, Algolia NeuralSearch, Marqo. *Build:* embed title+desc+attrs on save (observer); sqlite-vec local / Meilisearch vectors prod; same Scout-driven UI. — `meilisearch.com/blog/what-is-vector-search`
- **Visual search / search-by-photo** *(competitive)* — snap-a-photo → image-embedding KNN. Google Lens (20B/mo), Pinterest Lens, Amazon StyleSnap. *MY:* find modest-fashion/grocery by photo when EN/BM name unknown. *Build:* CLIP/SigLIP embeddings at upload → sqlite-vec/pgvector/Meilisearch; `Storefront\VisualSearch` upload + KNN. — `launchmystore.io/blog/visual-search-ecommerce-image-recognition`

## 3. Cart, checkout & promotions

- **Persistent multi-seller cart with adjustment lines** *(table-stakes)* — durable cart grouped per seller, promo/shipping/tax as adjustment lines (not price mutations). Medusa Cart, Saleor Checkout, commercetools. *Build:* group by seller; adjustment lines keep snapshots clean (Hard Rule 5). — `docs.medusajs.com/.../cart`
- **Checkout extensibility (validation/delivery/payment rules)** *(visionary)* — server hooks to validate cart, sort delivery, hide payment methods. Shopify Functions. *Build:* pipeline of PHP checkout-rule classes inside CheckoutService. — `shopify.dev/docs/apps/build/functions`
- **Programmable discount engine (BOGO/tiered/conditional free-ship/stackable)** *(table-stakes)* — Shopify Discount Functions, commercetools predicates, Vendure, Medusa Promotion. *Build:* condition/action pairs as cart adjustments; atomic consume under lock. — `docs.medusajs.com/.../cart/promotions`
- **Voucher/coupon codes with usage limits & eligibility** *(table-stakes)* — Saleor, Shopify, commercetools. *Already built (`VoucherService`).* — `docs.saleor.io/.../objects/voucher`
- **Free-shipping vouchers + voucher stacking** *(table-stakes for MY)* — Shopee model: one platform + one shipping + one shop voucher, fixed precedence, min-spend. *MY:* the single biggest conversion lever (5kg West / 1kg East free-ship). *Build:* typed `vouchers.kind` + `funded_by`; free-ship writes a `shipping_subsidy_sen` line. — `apps.apple.com/my/app/shopee-my-no-shipping-fee/id959841113`
- **Flash sales (time-boxed, stock-capped, countdown)** *(table-stakes for MY)* — Shopee Flash Deals, Lazada, Amazon Lightning. *MY:* 6.6/9.9/11.11/12.12 are the dominant demand spikes. *Build:* `flash_sales`+`flash_sale_items` (promo_price_sen, allocated_qty, per_buyer_limit); decrement allocation in the same lock as stock; `FlashSaleTimer` Livewire+Alpine. — `iprice.my/coupons/shopee/`
- **Group buy / share-to-unlock** *(competitive)* — group confirms when N join in a window; signed share URL is the unlock token. Pinduoduo, Shopee/Lazada Group Buy. *MY:* WhatsApp-native viral acquisition. *Build:* `group_buys`+`group_buy_members`; optimistic reserve, commit via status machine. — `flywheeldigital.com/blog/group-buying`
- **Gift cards & store credit** *(competitive)* — prepaid + issued credit redeemable at checkout; natural refund vehicle. Shopify, BigCommerce. *Build:* buyer ledger account in sen with code/balance; redeem as checkout adjustment under lock.
- **Gamification: spin-to-win, streaks, countdowns, live social-proof** *(competitive)* — Shopee Coins/check-in, Temu/Shein streaks; gamification market ~$19.4B 2025. *Build:* coin/streak ledger (integer balances) + VoucherService-backed spin-to-win issuing real vouchers; live counters via Livewire polling. Emerald = reward CTA, brass = ornament (Hard Rule 8). — `easyappsecom.com/guides/ecommerce-gamification-guide`

## 4. Orders, fulfilment & returns

- **Multi-seller sub-order splitting + lifecycle state machine** *(table-stakes)* — per-seller sub-orders, validated transitions, audit history. Amazon, Shopee, Medusa. *Already built (`SubOrderStatusService`).* — `docs.medusajs.com/.../commerce-modules`
- **Returns, refunds, exchanges & claims (RMA)** *(table-stakes)* — line-item-scoped returns with reason codes, approval, restock, money-back/exchange. Medusa returns/exchanges/claims, Amazon, Shopee. *MY:* Shopee 15-day Guarantee + dispute flow precede escrow release. *Build:* `return_request_items` (item, qty, refund_sen); refund via ledger; conditional atomic restock. — `docs.medusajs.com/.../commerce-modules`

## 5. Inventory & shipping

- **Atomic stock decrement & oversell prevention** *(table-stakes)* — validate+consume under row lock. *Already built (Hard Rule 3).*
- **Multi-location inventory + reservations + order routing** *(competitive)* — stock per location, reserved at checkout, routed by proximity/availability. Shopify, Saleor, Medusa. *Build:* `(variant_id, location_id)` inventory; reserve in the checkout lock; routing service by stock + buyer state.
- **Shipping zones, weight/price rates & carrier-calculated rates** *(table-stakes)* — per-zone + real-time carrier rates by weight/dimension/destination. Shopify, BigCommerce, ShipperHQ. *MY:* West vs East splits, weight tiers. *Build:* `ShippingService` with zone tables + courier-rate adapter; per sub-order. — `help.shopify.com/.../third-party-carrier-calculated-shipping`
- **Multi-courier aggregator (EasyParcel)** *(table-stakes for MY)* — one API to compare/book across J&T, Pos Laju, City-Link, Ninja Van, Flash, DHL; rate-check, AWB label, tracking webhook, sandbox + PHP wrapper. *MY:* how most SME merchants ship. *Build:* `EasyParcelDriver`; rate-check at checkout, book on order, webhook → SubOrderStatusService. — `developers.easyparcel.com/`, `github.com/apih/easyparcel`
- **Direct courier integrations (Ninja Van, J&T)** *(competitive)* — better rates/SLAs at volume; Ninja Van ninjaAPI (OAuth2 + webhooks), J&T Custcode/API key. *Build:* keep `ShippingService` driver-based so couriers are interchangeable. — `sites.google.com/ninjavan.co/ninjaapi/api-docs`
- **Free-shipping programs & thresholds** *(competitive)* — platform/seller-funded with who-pays accounting. Shopee weight-capped free-ship. *Build:* shipping-voucher type with funding source; negative shipping adjustment; ledger attributes the subsidy. — `malaymail.com/.../shopee-pos-malaysia-free-shipping`

## 6. Payments

- **Gateway with idempotent server-verified callbacks** *(table-stakes)* — trust the S2S callback + requery; browser redirect UX-only. Stripe webhooks, iPay88 BackendURL. *Already built (Hard Rule 4).* — `docs.stripe.com/`
- **FPX online banking (DuitNow/PayNet)** *(table-stakes for MY)* — real-time bank debit across 40+ MY banks; the dominant online method. *Build:* a bank-code parameter within the existing iPay88 flow; persist selected bank on the payment. — `hitpayapp.com/my/blog/payment-gateway-malaysia`
- **DuitNow QR + DuitNow Transfer** *(table-stakes for MY)* — national interoperable QR across banks + e-wallets. *Build:* expose via aggregator; reuse idempotent callback. — `main.billplz.com/blog/.../local-payment-methods`
- **E-wallets: Touch 'n Go, GrabPay, Boost, ShopeePay** *(table-stakes for MY)* — bundled within one iPay88/Billplz merchant account. *Build:* data-driven `payment_methods` enum; route through aggregator (no per-wallet SDK). — `ppro.com/news/ppro-grabpay-touch-n-go-malaysia/`
- **Cards (Visa/Mastercard) + 3-D Secure + tokenised saved cards + Apple/Google Pay** *(table-stakes)* — Razer added Apple Pay for MY. *Build:* hosted/tokenised flow only (never store PANs); store gateway token for saved cards. — `press.razer.com/.../apple-pay-malaysia`
- **BNPL: Atome, SPayLater, GrabPayLater** *(competitive)* — 3-instalment or credit line; ~97% of MY BNPL via Shopee/Grab/Atome. *Build:* enable via aggregator (same callback contract) or wrap Atome Payment API v2 as `AtomeService`; full amount captured, instalment risk on provider. — `doc.apaylater.com/v2/`, `rankpage.com.my/fintech/top-10-bnpl-providers-in-malaysia/`
- **Cash on Delivery with risk controls** *(table-stakes for MY)* — courier-collected, remittance reconciliation. *Already built; reconcile via ledger on remittance.* — `parceldaily.com/blog/guide-to-e-commerce-shipping-in-malaysia/`
- **Digital wallet / stored-value + coins** *(competitive)* — in-app balance + earn/burn loyalty coins. ShopeePay + Shopee Coins. *Build:* buyer wallet + coins as separate ledger accounts in sen; apply as checkout adjustment under lock. — `miracuves.com/blog/what-is-shopee-and-how-it-work/`
- **Seller wallet, ledger & payouts** *(table-stakes)* — append-only double-entry, commission/refund/withdrawal, payout scheduling. Stripe Connect, Shopee wallet. *Already built (`LedgerService`).* — `stripe.com/connect`
- **Commission resolution by category/seller/campaign** *(table-stakes)* — snapshot take-rate per sub-order. *Already built (`CommissionResolver`).*
- **Escrow / held-funds release on delivery** *(competitive — owner-selected)* — payment held, released on delivery-confirm or guarantee-window lapse. Shopee Guarantee, Stripe Connect delayed payouts. *MY:* THE trust mechanism. *Build:* ledger pending→available on `auto_complete_at`. — `bigseller.com/.../shopee-guarantee-period.htm`
- **Fraud / risk scoring at checkout** *(competitive)* — device/IP/velocity/behaviour → allow/review/block; Stripe Radar cuts fraud ~38%. *Build:* rule-based `RiskService` inside CheckoutService (velocity, voucher-abuse caps, COD blacklist, new-account limits) → admin review queue; `DeviceGuard`/Turnstile already present. — `stripe.com/radar`
- **Embedded finance / seller financing** *(competitive)* — BNPL at scale + lender-funded seller cash advances in-dashboard. Shopify Capital, Shopee SPayLater (Monee Capital). *Build:* seller-finance module reusing `LedgerService` for advances/repayments. — `seller.shopee.com.my/edu/article/21112`

## 7. Tax & compliance (Malaysia + worldwide)

- **Automated tax (SST/GST/VAT) with rules & exemptions** *(table-stakes)* — destination/origin rates, category classes, registration thresholds, reverse charge, zero-rating. Stripe Tax (100+ countries), Medusa/commercetools tax modules. *MY:* SST = Sales Tax 5%/10% + Service Tax 6%/8%; registration ~RM500k; many staples exempt; full enforcement 1 Jan 2026. Tax depends on each *seller's* registration + product class. *Build:* `TaxService` with jurisdiction resolver + `TaxClass` enum; per-line in sen; snapshot rate+flag onto order_items (Hard Rule 5); surface as its own line. — `cleartax.com/my/en/sst-in-malaysia`, `stripe.com/tax`
- **Imported Low-Value-Goods (LVG) 10% sales tax** *(competitive)* — 10% on imported online goods ≤ RM500 since 1 Jan 2024; RM500k registration trigger. *Relevant only if cross-border/dropship sellers ship sub-RM500 imports.* *Build:* `origin_country` flag; 10% line on item value (exclude shipping); track rolling 12-month LVG turnover. — `mylvg.customs.gov.my/Home`
- **LHDN MyInvois e-invoicing mandate** *(table-stakes for MY)* — phased by revenue band (Phase 4 RM1–5M from 1 Jan 2026, relaxation to 30 Jun 2026; sub-RM1M exempt per Dec 2025 Cabinet); each cleared invoice gets a UIN + validation QR. *Platform issues on behalf of sellers as Intermediary.* *Build:* `einvoice_required`/phase per store; `EInvoiceService` reading sacred order snapshots, fired on verified-paid. — `cleartax.com/my/en/different-phases-implementation-timelines-einvoicing-malaysia`
- **Consolidated B2C e-invoice + RM10,000 rule** *(competitive)* — monthly consolidated submission for un-requested B2C receipts (within 7 days of month-end), but each order > RM10,000 needs its own; buyer can always request one. *Build:* two paths — on-demand individual (or when `order_total_sen > 10_000_00`) + scheduled monthly consolidator; idempotent (store submission UID). — `jomeinvoice.my/latest-malaysia-e-invoicing-faqs/`
- **MyInvois API (submit/validate/cancel)** *(competitive)* — OAuth2 (Intermediary login for filing-for-many), Submit signed UBL 2.1/JSON, Get Document Details (Valid/Invalid + metadata), Cancel within 72h; preprod sandbox provided. *Build:* `MyInvoisProvider` wraps Laravel Http; cache OAuth2 token; build UBL from snapshots; digital signature (new cert dependency); poll then persist UIN+long-id; sandbox base URL until cutover. — `sdk.myinvois.hasil.gov.my/api/`

- **Loyalty: points, tiers & referrals** *(competitive)* — earn/redeem, VIP tiers, referral attribution. Smile.io, Yotpo; Shopee Coins + check-in. *Build:* coins/points ledger account; tier from rolling spend; redeem as checkout adjustment. — `apps.shopify.com/smile-io`
- **Daily check-in & gamification (Shopee Farm/Shake/spin)** *(visionary)* — habit-loop mini-games rewarding coins/vouchers; major DAU driver in MY. *Build:* `check_ins` + streak + daily reward listener crediting the coin ledger. — `malaymail.com/.../shopee-cny-coins`
- **Abandoned-cart recovery & lifecycle messaging** *(competitive)* — idle-cart reminders, back-in-stock, price-drop. - **Affiliate / creator commerce** *(competitive)* — trackable links, last-click attribution, payable commission, dashboards; Shopee opened to all channels, Lazada committed US$100M. *Build:* `affiliates`+`affiliate_links`+click/conversion tables; payable via ledger separate from CommissionResolver. — `thinkresonate.com/commerce-economy/affiliate-programme-shopee-lazada-tiktok`
- **B2B company accounts, price lists & net terms** *(visionary)* — companies with contacts, assigned catalogs, negotiated prices, Net-15/30/60. ## 10. AI-native & emerging (2025–26)

- **Conversational chat-to-shop concierge** *(competitive — now table-stakes at giants)* — NL assistant queries catalog, compares, deep-links PDPs. Amazon Rufus (250M users), Walmart Sparky, Shopify Sidekick, Perplexity Shopping. *MY:* bilingual EN/BM + halal/ingredient Q&A is a real differentiator. *Build:* `Storefront\ShopAssistant` Livewire → Claude (haiku, escalate to opus) with tool-use (`search_products`/`get_variant`/`check_stock`) over Scout+Eloquent; stream via `wire:stream`; live data only (Hard Rule 5). — `getpassionfruit.com/blog/how-to-use-claude-for-ecommerce`
- **Agentic checkout / external-agent buy protocols (ACP, AP2, UCP)** *(visionary)* — third-party AI agents discover products and purchase programmatically. OpenAI+Stripe ACP (live Sep 2025), Google UCP+AP2, Visa Trusted Agent, Mastercard Agent Pay. *MY:* early, but a compliant product feed + checkout intent endpoint positions sellers for discovery. *Build:* read-only ACP-style product feed (JSON from Scout) + a server-validated agent-checkout-intent reusing CheckoutService; trust idempotent server confirmations only. — `stripe.com/newsroom/news/stripe-openai-instant-checkout`
- **Agentic auto-buy / price-drop watch** *(visionary)* — buys autonomously when a target price is hit. Amazon Rufus auto-buy (Nov 2025). *MY:* fits mega-sale deal-hunting. *Build:* `price_watches` (target_price_sen) + scheduled job running CheckoutService under lock with a pre-authorised method and per-watch limits. — `novadata.io/.../amazon-rufus-agentic-auto-buy`
- **Real-time per-user personalization** *(competitive)* — in-session re-ranking + recommendation tiles from behaviour. Algolia (1–20ms), Constructor, Shopee 'For You'. *Build:* extend `RecommendationService` from rules to behavioural; log view/add/purchase events; co-visitation/co-purchase job for FBT; embeddings for similar-items. — `constructor.com/blog/ecommerce-personalization`
- **AI product-content generation (descriptions, SEO, alt text, translations)** *(competitive)* — bulk multi-language listing copy. Shopify Magic (~15 hrs/wk saved). *MY:* auto EN+BM (+ZH later) so micro-sellers list once. *Build:* seller "Generate with AI" → Claude (haiku) writing translatable fields (`en` always), draft for review. — `pagefly.io/blogs/shopify/shopify-magic`
- **AI image generation / lifestyle / variant shots** *(competitive)* — background cleanup + lifestyle composites + per-variant imagery. Shopify Magic image editing, Pebblely/Flair. *Build:* diffusion image model (NOT Claude — flag dependency) pipeline → spatie/medialibrary; Claude vision QAs/captions; human-approved. — `shopify.com/blog/ai-image-generator`
- **AI virtual try-on (diffusion)** *(visionary)* — render garment onto shopper photo. Google Virtual Try-On (2025), Wanna, 3DLook; apparel = 56% of returns. *MY:* high impact for modest-fashion/footwear. *Build:* hosted try-on service (flag dependency); async render job from PDP; start one category, measure return-rate. — `seosherpa.com/google-virtual-try-on/`
- **3D / AR product viewer** *(competitive)* — `<model-viewer>` GLB (Android/WebXR) + USDZ (iOS Quick Look); "view in your room". *Build:* pure front-end — `<model-viewer>` in a PDP partial, serve .glb/.usdz via medialibrary; no backend service. — `developers.google.com/ar/develop/webxr/model-viewer`
- **AI dynamic pricing & repricing** *(visionary)* — ML adjusts price from demand/competition/stock; Amazon ~2.5M decisions/day. *MY:* transparency matters (avoid surge perception). *Build:* `SellerPricingService` proposes (not auto-applies) `suggested_price_sen` the seller approves; money stays integer sen; snapshots immutable. — `xenoss.io/blog/ai-powered-dynamic-pricing`
- **AI demand forecasting & smart reorder** *(competitive)* — predict demand, set reorder points; accuracy 55–65% → 85–92%. Shopify forecasting, Toolio. *MY:* plan around CNY/Raya/mega-sale spikes. *Build:* seller widget; scheduled job computes velocity+seasonality → reorder-by date + qty; moving-average baseline, ML in prod. — `shopify.com/blog/ai-demand-forecasting`
- **AI fraud / risk scoring** *(competitive)* — see Payments §6 (`RiskService`). — `stripe.com/resources/more/ai-for-fraud-detection`
- **Live commerce / shoppable livestream** *(competitive)* — host-led video with "Buy Now" overlays, flash drops; SEA social commerce $47.6B in 2025; Shopee Live +37x views, +70% flash conversion. *MY:* near-mandatory channel. *Build:* MVP embeds external/HLS player in `Storefront\Live` Livewire + pinned product cards reusing CartService; live count/"just sold" via polling; checkout atomic; native video later (flag dependency). — `digitalinasia.com/2026/03/23/social-commerce-southeast-asia-revolution/`
- **Predictive replenishment / subscribe-and-save** *(visionary)* — auto-reorder staples on a cadence; Alexa+ free to Prime Feb 2026. *MY:* repeat halal grocery/household. *Build:* `subscriptions` (variant, cadence, next_run_at, price snapshot) + scheduled job via CheckoutService; each order self-snapshots. — `stellagent.ai/insights/agentic-commerce-grocery-predictive-replenishment`
- **Sustainability & circular commerce** *(visionary)* — carbon-neutral shipping, eco-packaging, resale/trade-in; recommerce → $291.6B by 2029. eBay/IKEA/Levi's resale. *MY:* growing green segment + "preloved" category. *Build:* optional carbon-offset shipping line (real sen surcharge) + `condition=preloved` flag + trade-in listing flow reusing product/order plumbing. — `bstock.com/blog/recommerce-...`
